# Mobile-Manufacturer-Data-Analysis.
SQL Advanced Case Study – Cellphone Sales Analysis
This project contains solutions to an advanced SQL case study based on a Cellphone Sales Database. The objective is to analyze sales transactions and generate meaningful business insights using advanced SQL concepts.

Project Overview

The database consists of the following tables:

Dim_Manufacturer
Dim_Model
Dim_Customer
Dim_Location
Fact_Transactions

The queries answer real-world business questions related to customer behavior, product performance, manufacturer sales, pricing analysis, and year-over-year trends.

Key SQL Concepts Used
Joins (INNER, LEFT)
Common Table Expressions (CTEs)
Aggregate Functions
GROUP BY and HAVING
Window Functions (RANK, DENSE_RANK, ROW_NUMBER, LAG)
CASE Statements
Subqueries
Date Functions
Ranking and Top-N Analysis
Percentage Change Calculations
Business Problems Solved
Identified states with cellphone purchases from 2005 onward.
Determined the US state purchasing the highest number of Samsung phones.
Analyzed transaction counts by model, ZIP code, and state.
Found the cheapest cellphone model available.
Calculated average prices for models from the top-selling manufacturers.
Identified high-value customers based on average spending.
Determined models consistently ranked among the top-selling across multiple years.
Compared top manufacturers across different years.
Identified manufacturers with new sales activity in 2010.
Performed customer spend analysis with year-over-year percentage change.
Skills Demonstrated

This project demonstrates strong SQL skills for data analysis, business intelligence, and reporting by solving practical business scenarios using efficient and optimized queries.
